# Plan: CC-11

## Work Item
**Title:** Taste Agent

## Overview
CC-11 implements a taste agent that personalises cocktail suggestions using the user's preferences and cocktail history — both of which are currently read by the orchestrator but never passed to any agent. At suggestion time, the agent filters out cocktails containing disliked ingredients and boosts scores for cocktails similar to highly-rated past drinks. After a cocktail is rated, a second entry point infers updated taste patterns from the log and writes them back to Preferences.md. This requires a new `write_preferences()` vault writer as infrastructure.
