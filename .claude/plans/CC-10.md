# Plan: CC-10

## Work Item
**Title:** Shopper Agent

## Overview
CC-10 implements a shopper agent that classifies missing ingredients by store accessibility tier — tier 1 (any supermarket), tier 2 (larger supermarket), tier 3 (specialist) — and recommends a "stock up" shopping list of tier 1 items that unlock the greatest number of recipes. The agent follows the same Gemini-backed pattern as the bartender agent and will be wired into the orchestrator chain after bartender suggestions are generated.

### Phase 2: Shopper agent
Implement the shopper agent following the bartender agent pattern. The agent uses a Gemini system prompt to classify each missing ingredient by accessibility tier, then applies deterministic logic to rank tier 1 items by the number of recipes they would unlock.

- [X] Write `src/agents/shopper.py` with `SYSTEM_PROMPT` and `_USER_PROMPT_TEMPLATE`
- [X] Implement `_extract_missing_ingredients(suggestions)` to deduplicate missing ingredients across bartender suggestions and count how many recipes each one appears in
- [X] Implement `_parse_tiers(response_text)` to extract structured JSON from the Gemini response (same regex-fence pattern as bartender)
- [X] Implement `suggest(suggestions: list[dict]) -> dict` as the public entry point — calls Gemini to classify tiers, then ranks tier 1 items by recipe-unlock count and returns a structured result with keys `tier_1`, `tier_2`, `tier_3`, and `stock_up` (sorted tier 1 items)
