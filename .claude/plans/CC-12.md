# Plan: CC-12

## Work Item
**Title:** Fix bartender test mock path

## Overview
CC-12 fixes four failing tests in `tests/test_bartender.py` caused by an incorrect mock access path. The Gemini client is called via `_client.chats.create(...)`, but the tests were asserting against `mock_chat.create` — omitting the `.chats` intermediate attribute — so `call_args` was always `None` and the assertions either raised `TypeError` or silently passed against the wrong mock. The fix updates all four affected assertions to use `mock_chat.chats.create`.
